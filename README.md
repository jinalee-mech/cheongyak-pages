# 청약 나침반

청약홈 공공데이터로 **"이번 주 분양 → 내 가점 → 이 지역 당첨선"** 세 단계만 보여주는 도구 — 기말 프로젝트.

라이브 페이지: https://깃허브아이디.github.io/저장소이름/  ← Pages 설정 후 본인 주소로 수정

## 무엇을 하나 (딱 3단계)

1. **이번 주 분양 중인 청약 목록** — 분양정보 API에서 현재 모집 중인 공고를 받아
   단지명·지역·면적·일정으로 보여주고, 유형 태그(민영 일반공급 / 공공 / 오피스텔 / 임대)와
   함께 **가점이 적용되는 공고인지**를 한눈에 표시.
2. **내 가점 계산** — 무주택기간·부양가족·통장가입기간을 입력하면 **84점 산식**으로 계산.
   단, **민영 APT 일반공급(가점제)** 공고에만 적용하고, 추첨·특공·공공·오피스텔이면
   "여긴 가점이 아니라 별도 기준(추첨/자격요건/납입금)"이라고 안내만 함.
   무주택기간·부양가족은 자동 추정하지 않고 입력값만 사용.
3. **이 지역, 최근엔 몇 점에 됐나** — 당첨 가점 통계에서 같은 지역의 과거 당첨선을
   "보통 ○○~○○점대" 밴드로 보여줌. 단지별 정밀 커트라인이 아니라 **지역 단위 참고치**이고,
   내 점수가 밴드 어디쯤인지 막대에 표시해 현실성을 가늠.

## 동작 구조 (서버 없음)

- [scripts/collector.py](scripts/collector.py) 가 GitHub Actions에서 주기적으로 청약홈 API를 호출해
  저장소 루트의 `data.json` 을 갱신합니다.
- [index.html](index.html) 는 그 `data.json` 을 그대로 fetch 해서 그립니다. 서버가 필요 없습니다.
- `SERVICE_KEY` 가 없으면 collector는 `data.json` 을 건드리지 않고, 페이지는 **데모 데이터(예시 단지)** 로 동작합니다.

## 실데이터 켜기

1. 공공데이터포털에서 인증키를 발급받아 저장소
   **Settings → Secrets and variables → Actions** 에서 이름 `SERVICE_KEY` 로 등록.
2. **Actions 탭 → `update-data` → Run workflow** 로 한 번 수동 실행. 성공하면 `data.json` 이
   자동 커밋되고, 페이지 헤더가 초록 "실데이터 · 수집 시각" 배지로 바뀝니다.

이후 매시 정각마다 자동 갱신됩니다. 주기는 [.github/workflows/update-data.yml](.github/workflows/update-data.yml)
의 `cron` 한 줄로 조절.

## 한계 (정직하게)

- "실시간"이 아니라 **주기 갱신**입니다. 헤더의 **수집 시각**이 기준입니다.
- **가점제/추첨제 비율은 공공데이터로 제공되지 않습니다.** 그래서 유형 태그는
  "민영 APT 일반공급이면 가점 적용" 수준의 분류이며, 같은 공고 안의 추첨 물량 비율까지는
  구분하지 못합니다. 정확한 적용 기준은 반드시 공식 공고문을 확인하세요.
- 3단계 밴드는 **지역 단위 통계**라 특정 단지의 커트라인과는 다를 수 있습니다.
